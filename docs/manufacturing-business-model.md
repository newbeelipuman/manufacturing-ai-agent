# 制造业业务模型设计

## 1. 项目定位

本项目是制造业企业内部 AI Agent 后端 MVP，用模拟 ERP/MES/WMS 数据验证安全可控的只读查询闭环。系统不接真实企业生产系统，不执行审批、出库、调账、下单等写侧业务动作。

## 2. 为什么不是简单 RAG 问答

项目不仅检索 SOP 文档，还会识别订单、工单、采购单、SKU 等业务对象，调用只读工具查询模拟表数据，经过权限校验、业务风险判断、答案组合、审计日志和 usage 统计后返回结果。

## 3. 核心业务对象

- 销售订单：客户需求、交付日期、交付状态和订单行。
- 库存 SKU：可用库存、锁定库存、质量冻结库存和单位。
- 库存批次：批次号、仓库、可用数量、锁定数量和有效期。
- 工单：计划数量、完工数量、计划开工日期和所需物料。
- 采购单：供应商、预计到货日期、延期状态和采购明细。
- SOP 知识：异常处理、交期沟通、缺料处理等文档片段。

## 4. 业务对象之间的关系

- 订单依赖 SKU 可用库存和批次状态。
- 库存受锁定库存和质量冻结库存影响。
- 工单影响未来补货和开工齐套判断。
- 采购到货影响缺料缓解和订单交付风险。
- SOP 支撑异常处理建议，但不替代人工确认。

## 5. 订单发货风险模型

业务问题：订单 O1001 当前能否发货。

输入数据：销售订单、订单行、SKU 库存、采购到货、交期异常 SOP。

判断逻辑：比较订单需求、可用库存、锁定库存和质量冻结库存，结合采购延期信号输出风险。

风险输出：`low`、`medium`、`high`、`blocked` 或 `unknown`。

人工复核边界：系统只给只读建议，任何出库、承诺客户交期或调账动作都必须由业务人员在企业系统中人工确认。

## 6. 工单开工齐套模型

业务问题：工单 WO1001 今天能否开工，缺哪些物料。

输入数据：工单、工单物料、物料库存、缺料处理 SOP。

判断逻辑：计算每个物料的剩余需求和可用库存差额，判断是否存在缺料。

风险输出：可开工、缺料高风险、未找到工单或权限阻断。

人工复核边界：系统不执行开工、领料或排产调整，只提示生产主管复核。

## 7. 采购延期影响模型

业务问题：采购单 PO1001 延期会影响哪些客户订单。

输入数据：采购单、采购明细、相关 SKU 的销售订单、采购延期沟通 SOP。

判断逻辑：按采购明细 SKU 关联销售订单，结合采购延期状态输出影响范围。

风险输出：高风险、低风险、未找到采购单或权限阻断。

人工复核边界：系统不修改采购承诺、不通知客户、不调整订单，只提供沟通和复核建议。

## 8. 权限与审计设计

系统角色包括 `admin`、`production_manager`、`sales`、`warehouse`、`purchase` 和 `normal_user`。每次工具调用前必须检查权限，允许和拒绝都会写入 `tool_call_log`。`/api/chat` 调用会写入 `agent_call_log`，并保存 intent、risk_level 和结构化响应快照。admin 可以查看审计日志和 usage 统计。

字段级 response filtering 在返回前执行，不改变数据库原始数据，也不改变工具内部查询逻辑。它用于演示不同角色可见字段不同的企业边界。

## 9. 当前 MVP 边界

- 当前不接真实 ERP/MES/WMS。
- 当前使用模拟数据。
- 当前不做业务写操作。
- 当前不做自动审批、出库、调账、下单。
- 当前不引入真实 LLM。
- 当前意图识别是规则。
- 当前实体抽取是正则。
- 当前 SOP 检索是关键词。
- 项目目标是验证企业 Agent 的安全可控查询闭环。

## 10. 后续可扩展方向

- 将模拟接口替换为真实 API、数据库视图或中间表。
- 引入更稳定的向量检索和文档版本管理。
- 扩展字段级权限策略和数据脱敏策略。
- 增加更完整的运营统计维度。
- 在保持只读边界的前提下接入真实企业身份体系。
