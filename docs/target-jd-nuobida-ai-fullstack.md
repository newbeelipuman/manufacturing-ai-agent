# 目标岗位 JD：诺必达 AI 全栈 / 应用工程师

## 基本信息

- 公司：东莞市诺必达智能科技有限公司
- 岗位：AI 全栈 / 应用工程师
- 地点：广东东莞
- 经验：1-3 年
- 学历：大专
- 薪资：15-30K
- 招聘人数：1-2 名
- 汇报对象：AI 部门负责人 / 总经理

## 岗位定位

和负责人一起，把大模型真正落到业务里，做能稳定上线、解决实际问题的 AI 系统，而不是停在 Demo。

## 核心工作

1. 落地企业 AI 转型：梳理业务场景，把 AI 方案一步步实现出来。
2. 开发企业级智能体 Agent：业务逻辑设计、工具调用、与企业现有系统对接。
3. 建设自研 AI 平台：参与开发与完善大模型网关 / 路由、Agent 编排、知识库 RAG、权限与安全、用量管理等核心模块。
4. 同时负责后端、部署与运维：把产品稳定地跑在服务器 / 私有云上，并持续维护。
5. 工作节奏：快速出原型、快速迭代、上线见结果。

## 必备要求

- 扎实的后端开发能力，Python / Node.js / Go 等其一即可，熟悉 Web 框架与 API 设计。
- 有 SaaS / 平台类产品的实际开发与上线经验。
- 熟悉服务器部署与运维：Linux、Docker、Nginx、CI/CD，云服务器或私有云环境。
- 熟悉数据库，关系型数据库必备，向量数据库优先。
- 对大模型应用开发有热情，了解或能快速上手 LLM API、Agent 框架、RAG、提示工程。
- 技术栈较新、学习能力强，能独立把问题解决掉。
- 有冲劲、扛得住事、结果导向。

## 加分项

- 做过 Agent / 智能体、工作流编排、知识库 / RAG 系统。
- 熟悉 Claude / OpenAI 等大模型 SDK 的二次开发，做过多模型网关 / 路由。
- 熟悉前端 React 等，能独立完成全栈交付。
- 有企业 ERP / 业务系统 API 对接、数据管道经验。
- 有 2B / 2C SaaS 从 0 到 1 经验。
- 有制造业 / 电商 / 跨境电商行业背景。

## 对当前项目的含义

当前 `manufacturing-ai-agent` 项目应优先强化以下方向：

- 企业 Agent 后端闭环，而不是泛用聊天机器人。
- 工具调用、权限、安全、审计、用量管理。
- RAG 的可解释证据链和评测报告。
- Docker、Nginx、私有化部署说明。
- 模拟 ERP/MES/WMS 对接边界，以及未来替换真实 API 的路径。
- 简洁可演示的 API 和报告，证明可以快速出原型并迭代。
